Plugins

Adding a new function

Can I add functions to SJQ? Yes, of course you can! This feature can be used to create plugins or just make life easier. It means that instead of doing this: mySJQFunction(elem,arg), you can do this: $mySJQElem.mySJQFunction(arg). And the best bit: It's super simple to use!

View the example below:

$.fn.mySJQFunction = function(arg) {
    // Content
    alert ('it works');
};

When you create a function (like mySJQFunction()) inside the $.fn object, you are allowing that function to be called from a SJQ object, so now we could run $mySJQElem.mySJQFunction(arg) and we would get a popup saying 'it works'.

If you wanted to access the SJQ object that the function is being run on, which is probably why you are doing this in the first place, use either $(this) or this. You probably want to return this.each(){/*your code here*/}; too, so that if it's running on something that there could be multiple of, it will loop through each one. You can use $(this) inside that to get the current object.

If you're running code that needs to be on the raw HTMLElement object, use .get(index). This will return a HTMLElement object, or an array of HTMLElements if no argument is supplied. If you were to look through the source code, you would notice this being used.
